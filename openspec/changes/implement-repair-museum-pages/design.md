## Context
需要实现两个核心页面：修复页面（/repair/:id）和博物馆页面（/museum）。修复页面需要支持交互式打卡、完成选择、墓碑封印动画等复杂交互。博物馆页面需要展示已归档的纪念品列表。

## Goals / Non-Goals
- Goals:
  - 实现完整的修复页面交互流程
  - 实现墓碑封印 CSS 动画（约 1200ms 后导航）
  - 实现博物馆页面列表展示
  - 确保已归档纪念品在修复页面为只读
- Non-Goals:
  - 不实现深度对话或心理咨询功能
  - 不实现云同步（仅 localStorage）
  - 不实现复杂的象征物动画（MVP 版本简化）

## Decisions
- Decision: 使用 CSS 动画实现墓碑封印效果
  - 理由：纯 CSS 动画性能好，无需额外库，符合 MVP 简洁原则
- Decision: 墓碑封印动画后约 1200ms 导航到 /create
  - 理由：给用户足够时间观看动画，同时保持流畅体验
- Decision: 已归档纪念品在修复页面禁用点按功能（只读模式）
  - 理由：防止用户误操作修改已归档的纪念品，保持数据完整性
- Decision: 象征物视觉反馈使用简单的进度比例计算
  - 理由：MVP 阶段先实现基础视觉反馈，后续可优化

## Risks / Trade-offs
- 风险：墓碑封印动画可能在不同设备上表现不一致
  - 缓解：使用标准 CSS 动画属性，测试主流浏览器
- 风险：localStorage 数据可能丢失
  - 缓解：MVP 阶段接受此限制，后续可考虑导出功能
- 权衡：简化象征物动画 vs 用户体验
  - 选择：MVP 优先功能完整性，动画可后续优化

## Migration Plan
- 无需迁移：这是新功能实现，不影响现有数据
- 数据兼容：新添加的 `archivedAt` 和 `releasedAt` 字段为可选，现有数据不受影响
